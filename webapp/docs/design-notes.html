Exploding Places Webapp Design Notes

Data Model

The server can host several games simultaneouly. Each is represented by a Game object, including field active (boolean). On creation each game is given a name and a specific ContentGroup to use. These are started and stopped using the orchestration/games.html web page.

Each ContentGroup represents a distinct set of game content, i.e. Zones and TimelineEvents. These are created by uploading an authored XML configuration file using the author/content.html page. This XML configuration file is created using the stand-alone authoring tool.

A Player is associated with a particular Game.

A Member is associated with a particular Player and Game.

A Message is associated with a particular Player.

Execution

On start-up the application caches Zone information (updated as new Zones are added/changed).

Every 5 seconds the container calls the game Engine's checkEvents method.

(Currently) this, for each active Game, gets all TimelineEvents with startTime since its last check and calls handleEvent on them.

This gets each Member in that Game and Zone (if zone-specific), and modifies its attributes as specified in the TimelineEvent.
