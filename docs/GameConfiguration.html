Game Configuration

Some notes on game configuration (GameConfig) options.

Game time rate is currently compiled in at a rate such that 1 minute = 200 game time units; in the authoring tool one timeline column = 100 time units, hence 30 seconds.

General Configuration:

  • startTime: game time when game becomes ACTIVE (i.e. "play" is pressed in orchestration interface). 
    • Should typically be negative or zero; typically no timeline events will occur while game time is negative, providing time for briefing users on the interface. 
    • E.g. 1 minute initial delay = -200
  • endTime: game time when the server will automatically change game state from ACTIVE to ENDING and send the game ending message to each player. 
    • E.g. 16 "years" (editor columns) = 1600 (~8 minutes or wall clock time)
  • contextMessageEndTitle & contextMessageEnd: title and test of "game ending" message sent to clients when game endTime is reached.  
Adding Stories:
  • enableAuthoring ==1 => server will enforce checks on client ability to author,
    • i.e. enableAuthoring = 0 => players can author at any time.
    • If enableAuthoring = 1 and enableAuthoringQuota = 0 then I don't think player can ever author
  • enableAuthoringQuota ==1 => server will run logic to change player ability to author according to whether a community member has brains >= 8
    • If enableAuthoring !=1 this will have no effect (I think)
Adding Members:

  • enableMemberQuota !=0 => enforce quota per member
  • initialMemberQuota [Integer, new], defaults to 1
Member Game Rules:
  • enableSoleMemberZoneEffect - not used?
